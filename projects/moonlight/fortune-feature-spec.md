# 학업 운세 기능 기획서

문라이트 대학 이벤트 — 새학기 응원 챌린지 활성화를 위한 부가 활동 기능.

---

## 배경

- 새학기 응원 챌린지 진행 중 (마감 3/13)
- 핵심 문제: 참여자 71%가 응원 0명 — "남에게 부탁"하는 구조의 한계
- 해결: 본인 활동으로 응원이 쌓이고, 활동 결과물이 자연스럽게 바이럴되는 구조로 전환

## 목표

1. **회원가입 증대** — 재밌는 콘텐츠로 대학 계정 가입 유도
2. **응원 카운트 활성화** — 활동 참여 = 응원, 공유 유입 = 응원
3. **자연스러운 바이럴** — 공유하고 싶은 카드 → 부탁 없는 확산
4. **매일 재방문** — 하루 1회 새 운세로 리텐션

---

## 콘텐츠: 새학기 학업 운세

새학기 분위기를 살린 학업 맥락 운세. MBTI 기반으로 "나를 읽어주는" 느낌.

### 입력

| 항목 | 필수 | 예시 |
|------|------|------|
| MBTI | 필수 | INTJ |
| 날짜 | 자동 (오늘) | 2026-03-03 |
| 전공 | 옵션 | 컴퓨터공학 |
| 학년 | 옵션 | 대학원 석사 |

### LLM 출력 (JSON)

```json
{
  "type": "계획은 완벽한데 실행은 내일부터 하는 타입",
  "fortune": "새학기 첫 주, 의외의 과목에서 영감을 받는 날. 수강신청 때 고민했던 그 과목이 정답일 수 있다.",
  "lucky_subject": "통계학",
  "lucky_color": "네이비",
  "lucky_item": "아이스 아메리카노",
  "reason": "새로운 시작에는 차가운 각성이 필요한 법"
}
```

### LLM 프롬프트 가이드

- **톤**: 가볍고 위트 있게. 공감 가는 대학생 일상. 새학기 설렘+긴장 반영
- **성향(type)**: MBTI 특성을 학업 상황에 대입. 웃기거나 찔리는 한 줄. 새학기 맥락 ("수강신청", "OT", "새 노트 정리", "첫 수업", "새 학기 다짐" 등)
- **운세(fortune)**: 2~3문장. 새학기 시작, 새로운 만남, 수업 분위기 등 시즌감 반영
- **행운 요소**: 대학생 일상에서 뽑기 (과목, 색, 아이템). 이유 한 줄은 위트 있게
- **매일 다른 결과**: 날짜를 시드로 활용. 같은 MBTI라도 날마다 다른 운세
- 전공/학년 입력 시 해당 맥락 반영 (예: "대학원생의 논문 운세", "새내기의 첫 수업 운세")

## 카드 디자인

한 장 카드. 인스타 스토리 / 카톡 공유에 최적화.

```

│   실행은 내일부터 하는 타입"   │

│                              │

│                              │
│  🍀 행운 과목  통계학          │
│  🎨 행운 색    네이비          │
```

- 하단에 문라이트 브랜딩 + "나도 해보기" CTA
- 동적 OG 이미지로 생성 (Vercel OG / Satori 권장)

## 사용자 흐름

### 참여자 흐름

```
대학 계정 로그인
  → 운세 페이지 진입
    → MBTI 입력 (최초 1회, 이후 저장)
    → (옵션) 전공·학년 입력
    → "오늘의 운세 받기"
      → LLM 생성 → 카드 렌더링
      → 응원 +1 (하루 1회)
      → "공유하기" 버튼 → 공유 링크 복사 (ref=내id)
```

### 유입자 흐름 (공유 링크)

```
공유 링크 클릭 (ref=공유자id)
  → OG 카드 미리보기 (카톡/인스타에서 카드 보임)
  → 랜딩 페이지: 공유자의 카드 + "나도 해보기" CTA
    → 대학 계정 가입/로그인
      → 공유자에게 응원 +1 (대학 계정만 유효)
      → 본인도 운세 참여 가능
```

## 응원 카운트 규칙

| 행위 | 대상 | 조건 | 횟수 |
|------|------|------|------|
| 운세 참여 | 본인 +1 | 대학 계정 로그인 | 하루 1회 |
| 공유 링크 유입 | 공유자 +1 | 유입자가 **대학 계정으로** 가입/로그인 | 유입자당 1회 |

- 일반 계정은 카운트 안 됨
- 1인 1카운트: 같은 유입자가 여러 번 들어와도 1회만

## 공유 링크

```
https://themoonlight.io/fortune?ref={userId}
```

- `ref` 파라미터로 공유자 추적
- 유입자 가입/로그인 완료 시 ref 유저에게 응원 +1
- OG 메타태그에 카드 이미지 URL 포함 → 카톡/인스타/스레드에서 카드 미리보기

## 바이럴 루프

```
A 운세 참여 → A 응원 +1 → 카드 공유
  → B 클릭 → 대학 가입 → A 응원 +1
    → B 운세 참여 → B 응원 +1 → 카드 공유
      → C 클릭 → 대학 가입 → B 응원 +1
        → ...
```

핵심: "응원해줘" 부탁이 "이거 재밌어 해봐" 공유로 바뀐다.

## 확장 (이후)

- 다른 활동 추가 (각각 하루 1회, 응원 +1): 시험운, 교수님 궁합, 학기 목표 뽑기 등
- 활동별 카드 디자인 차별화
- 응원 챌린지 페이지에서 활동 목록 노출 → 여러 활동 참여 유도
- 주간 랭킹 / MBTI별 통계 → 추가 공유 소재

## 기술 참고

- 카드 이미지: `@vercel/og` (Satori) — Next.js Edge Runtime에서 동적 OG 이미지 생성
- LLM: 구조화된 JSON 출력. 날짜+MBTI를 시드로 매일 다른 결과
- 대학 계정 판별: 기존 대학 메일 인증 체계 활용
- MBTI는 최초 입력 후 프로필에 저장, 이후 자동 적용
