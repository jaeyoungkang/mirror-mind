# 학업 운세 카드 — 추가 구현 프롬프트

기존 기획서(`fortune-feature-spec.md`) 기반 구현 완료 상태에서 카드 비주얼 업데이트.

---

## 1. MBTI 4기질별 배경 패턴

카드 배경을 MBTI 기질 그룹에 따라 4종으로 차별화한다.

| 기질 | MBTI | 배경 | 톤 |
|------|------|------|-----|
| NT (분석가) | INTJ INTP ENTJ ENTP | 기하학 그리드 + 퍼플 글로우 | 네이비/퍼플 |
| NF (외교관) | INFJ INFP ENFJ ENFP | 유기적 곡선 + 수채화 번짐 | 라벤더/코랄 |
| SJ (관리자) | ISTJ ISFJ ESTJ ESFJ | 정돈된 도트 패턴 + 그린 글로우 | 민트/그린 |
| SP (탐험가) | ISTP ISFP ESTP ESFP | 자유로운 스플래시 + 웜 글로우 | 오렌지/웜 |

- CSS 그라데이션 + 패턴으로 구현 (이미지 파일 불필요)
- OG 이미지(Satori)에서도 동일 패턴 적용
- 벚꽃잎(🌸) 장식 요소를 카드 곳곳에 투명도 낮게 배치 (새학기 분위기)
- 디자인 참고: `projects/moonlight/fortune-card-samples.html` 열어서 확인

## 2. 행운 요소 변경: 6종 풀 → LLM이 2개 선택

기존 3개 고정(과목/색/아이템) → **6종 풀에서 LLM이 매일 2개를 골라서 표시**.

### 6종 풀

| 키 | 라벨 | 이모지 예시 | 예시 값 |
|----|------|-----------|--------|
| `lucky_subject` | 행운의 과목 | 📊 🎭 📐 | 통계학, 심리학, 선형대수 |
| `lucky_item` | 행운의 아이템 | ☕ 📒 🎧 | 텀블러, 플래너, 이어폰 |
| `lucky_place` | 행운의 장소 | 🏫 📚 🌳 | 정문 카페, 도서관 3층, 잔디밭 |
| `lucky_snack` | 행운의 간식 | 🍙 🧁 🍫 | 삼각김밥, 붕어빵, 초코바 |
| `lucky_time` | 행운의 시간대 | 🕐 🌙 ☀️ | 오전 9시, 새벽 2시, 점심 직후 |
| `avoid` | 오늘 피해야 할 것 | ⚠️ | 교수님 질문, 조별과제 톡방, 학식 카레 |

### LLM 출력 변경

```json
{
  "type": "계획은 완벽한데 실행은 내일부터 하는 타입",
  "fortune": "새학기 첫 주, 의외의 과목에서 영감을 받는 날...",
  "lucky": [
    { "key": "lucky_subject", "emoji": "📊", "label": "행운의 과목", "value": "통계학" },
    { "key": "avoid", "emoji": "⚠️", "label": "오늘 피해야 할 것", "value": "교수님 질문" }
  ],
  "reason": "새로운 시작에는 차가운 각성이 필요한 법"
}
```

- `lucky` 배열은 항상 2개
- LLM이 그날 MBTI에 맞는 조합을 자유롭게 선택
- 프롬프트에 6종 풀 목록과 "2개를 골라라" 지시 포함

### 카드 UI에서 행운 요소 렌더링

- 2개를 나란히 가로 배치
- 이모지 크게 (32px), 값 텍스트 굵게 (15px bold)
- 세로 정렬: 이모지 → 라벨(작게) → 값(크게)
- 배경: 반투명 카드 (`rgba(255,255,255,0.07)`, border-radius 16px)

## 4. fortune_ref 보존 (중요)

공유 링크로 미가입자가 유입 → 가입 플로우 중 `fortune_ref` 파라미터가 소실될 수 있다.

- `FortuneReferralHandler`에서 `fortune_ref` 감지 시 `localStorage`에 저장
- 가입/로그인 완료 후 `localStorage` 체크 → `POST /referral-cheer` 호출 → 삭제
- 이 로직 없으면 바이럴 루프의 핵심 고리(공유→유입→공유자 응원+1)가 끊긴다
