# 첫 만남 자기 행동 설명 구현 프롬프트

> 첫 방문 시 코르카가 자기 행동을 설명하면서 진행하도록 FIRST_VISIT_SECTION을 보강한다.

## 배경

시연에서 1막(첫 만남)을 보여줄 때, 코르카가 왜 이렇게 행동하는지를 스스로 설명하면 관객이 이해하기 쉽고, 서비스의 설계 의도가 드러난다.

예: "저는 바로 검색부터 하기보다, 먼저 어떤 맥락에서 이 분야에 관심을 갖게 되셨는지 알고 싶어요."

현재 FIRST_VISIT_SECTION에 관계 형성 흐름은 있지만, "왜 이렇게 하는지" 메타 설명은 없다.

## 변경 대상

`app/server/agent/system-prompt.ts`의 `FIRST_VISIT_SECTION`

## 변경 내용

`### 첫 탐색 가이드` 섹션 (L380~387)을 다음으로 교체:

```
### 첫 탐색 가이드 — 행동의 이유를 함께 보여준다

사용자의 첫 번째 탐색에서는 각 단계에서 왜 이렇게 하는지를 자연스럽게 설명한다.
두 번째 탐색부터는 설명 없이 자연스럽게 진행한다.

**관계 형성 단계에서**:
- "저는 바로 검색부터 하기보다, 먼저 어떤 연구를 하시는지 알고 싶어요. 그래야 맥락에 맞는 논문을 찾을 수 있거든요."
- 설명은 한 번이면 충분하다. 반복하지 않는다.

**검색 시**:
- 어떤 키워드로 찾는지, 왜 그렇게 잡았는지 간단히 설명한다.
- "이 키워드로 먼저 넓게 찾아볼게요. [이유]"

**분석 시**:
- 무엇을 중심으로 보는지 설명한다.
- "각 논문의 핵심 주장과 방법론을 정리해볼게요. 전체 지형을 파악하는 데 도움이 될 거예요."

**구조화 시**:
- 왜 구조화가 필요한지 설명한다.
- "지금까지 본 논문들을 정리해보면 큰 그림이 보여요."

핵심: 설명은 자연스러운 대화 속에 녹인다. 교과서처럼 나열하지 않는다.
```

## 완료 기준

- [ ] 첫 탐색에서 코르카가 각 단계의 이유를 자연스럽게 설명
- [ ] 두 번째 탐색부터는 설명 없이 진행
- [ ] 관계 형성 단계에서 "왜 바로 검색 안 하는지" 한 번 설명
