## 탐색 전략 유형

### Berry Picking (Bates, 1989)
키워드를 계속 바꿔가며 반복 탐색. 찾을 때마다 새 방향이 열린다.
- **언제**: 새 분야 진입, 탐색적 질문, 연구 질문이 아직 형성 중일 때
- **특징**: 미리 계획 불가. 발견이 다음 검색을 결정. 과정 자체가 자기 지시적
- **예**: "transformer diversity" 검색 → contrastive learning 발견 → "contrastive recommendation" 재검색 → ...

### Snowballing / Citation Chasing
한 논문의 참조(backward) + 인용(forward)을 따라가며 확장.
- **언제**: 핵심 논문 발견 후 관련 연구 확장
- **특징**: 키워드 검색이 놓치는 49%를 커버. 체계적 리뷰에서 최종 참조의 51% 차지
- **두 방향**: backward(이 논문이 뭘 참조했나) + forward(누가 이 논문을 인용했나)

### Pearl Growing
"씨앗 논문" 하나에서 키워드+인용을 동시에 확장하는 구조적 접근.
- **언제**: 완전성이 필요한 리뷰
- **특징**: snowballing보다 구조적. 포화(새 논문이 안 나올 때까지) 반복

### Scoping Review
넓게 훑어 지형 파악. 체계적이지만 엄격하지는 않다.
- **언제**: 분야 진입, 연구 제안서 준비, 체계적 리뷰 전 사전 탐색
- **특징**: 반복적, 포용적, 빠름. 시나리오 1, 2에 가장 적합

### Systematic Review
프로토콜 기반, 재현 가능, 모든 관련 논문 포착.
- **언제**: 특정 질문에 대한 완전한 답이 필요할 때
- **특징**: 10개월 이상 소요. 2인 이상 필수. 편향 최소화

## 연구자 행동 모델 (Ellis)

1. **Starting** — 시작점 찾기. 동료 추천 > 지도교수 > DB 검색
2. **Chaining** — 참조/인용 따라가기
3. **Browsing** — 반구조적 탐색 (학회 목차, 프리프린트 스캔)
4. **Differentiating** — 출처 필터링 ("Nature급만", "최근 2년만")
5. **Monitoring** — 현행 인지 (알림, 학회, 동료 소식)
6. **Extracting** — 선별적 추출 (초록+결론만 읽기)

## 핵심 시나리오

### 시나리오 1: 새 분야 진입

**누가**: PhD 학생, 분야 전환 연구자
**기간**: 3~6개월
**흐름**:
1. 지도교수/동료가 핵심 논문 3~5편 추천 (Starting)
2. 불안과 혼란 — "내가 뭘 모르는지 모른다" (Kuhlthau: Uncertainty)
3. 추천 논문 읽으며 backward snowballing (Chaining)
4. **전환점**: "아, 이 분야에 3가지 하위 문제가 있구나" (Focus Formulation)
5. Forward snowballing으로 최근 연구 파악
6. 자기만의 연구 질문 형성
7. 타겟 검색으로 전환

**핵심 고충**:
- 키워드를 몰라 검색 자체가 안 됨
- 분야의 전체 지형을 파악하기까지 수주~수개월
- 중요한 논문과 아닌 것을 구분할 안목이 부족

**동료가 도울 수 있는 것**:
- "이 분야의 핵심 논문은 이것들이에요" (evaluate_importance의 진화)
- "이 논문의 참조 중 이런 것들이 기초적이에요" (backward snowballing 지원)
- "지금까지 읽은 것을 보면, 3가지 방향이 보여요" (구조화)

### 시나리오 2: 연구 제안서/그랜트 준비

**누가**: 교수, 연구원, 포닥
**기간**: 3~4주
**흐름**:
1. "내 제안이 채울 갭을 찾아야 한다" (명확한 목표)
2. 3개 DB에서 키워드 검색 → 200~400편 히트
3. 초록 기반 빠른 트리아지 → 30~40편 다운로드
4. 8~10편 핵심 논문 선정 + 개념적 분류 (방법론별, 접근별)
5. Snowballing으로 40~60편까지 확장
6. 갭 발견: "기존 연구는 전부 X를 가정, Y는 미탐색"
7. 갭을 연구 제안의 근거로 사용

**핵심 고충**:
- 빠르게 지형을 파악해야 함 (시간 압박)
- 용어 변이로 관련 논문을 놓침
- 갭이 진짜 갭인지 판단하기 어려움

**동료가 도울 수 있는 것**:
- 대량 검색 결과에서 빠른 트리아지 (analyze + evaluate)
- 용어 변이 감지: "이 주제는 다른 이름으로도 연구되고 있어요"
- 갭 분석: "기존 연구의 공통 한계를 정리하면 이렇습니다"
- 개념적 분류: cluster를 연구 방향이 아닌 방법론/접근법 기준으로

### 시나리오 3: 놀라운 논문 발견 후 맥락 파악

**누가**: 모든 연구자
**기간**: 30분~1시간
**흐름**:
1. 알림이나 우연으로 기존 가정을 뒤집는 논문 발견
2. 초록+결론 읽기 (5분)
3. 저자 신뢰도 확인
4. Backward: 이 논문의 기반은? (10분)
5. Forward: 이미 반응한 사람이 있나? (10분)
6. 동료에게 공유, 의견 수집

**동료가 도울 수 있는 것**:
- 빠른 분석: "이 논문의 핵심 주장과 근거를 정리했어요"
- 맥락 제공: "이 논문이 인용한 것들 중 당신이 이미 본 것이 있어요"
- 반응 검색: "이 논문을 인용한 최근 논문 N편을 찾았어요"

### 시나리오 4: 연구 트렌드 파악

**누가**: 분야 관찰자, 그랜트 심사자, 분야 진입자
**흐름**:
- 키워드 빈도 분석 (시간축)
- 인용 분석 (영향력 논문)
- 클러스터링 (주제 그룹)
- 성장 방향 vs 쇠퇴 방향 식별

**동료가 도울 수 있는 것**:
- 연도별 키워드 추이 시각화
- "이 방향은 2023년부터 급성장 중이에요"
- "이 클러스터는 최근 논문이 줄고 있어요"

### 시나리오 5: 현행 유지

**누가**: Senior 연구자
**핵심 고충**: 논문이 12년마다 2배 증가. 관련 논문의 30~50%를 놓침.
**현재 방법**: TOC 알림, 핵심 연구자 팔로우, 학회, 랩미팅, 프리프린트 스캔

**동료가 도울 수 있는 것**:
- "지난주 당신의 연구 주제 관련 새 논문 N편이 나왔어요"
- "그중 이것은 Chen 2024의 후속 연구로 보여요"
- 알림 역할의 내재화

## 논문 관리의 실제 행동

### 트리아지 (가장 큰 노동)
- 200편 검색 → 30편 다운로드 → 10편 정독 → 3편 핵심
- 판단 기준: 초록, 저자, 학회, 인용수, 년도
- **논문 28%만** 도구로 관리. 40% 교수가 아무 도구 안 씀

### 맥락 소실 (가장 큰 고충)
- 논문을 왜 저장했는지 6개월 뒤 잊어버림
- 메모는 현재 이해용. 나중에 다시 볼 때 맥락이 없음
- "이 논문이 내 연구에 왜 중요했지?"가 가장 흔한 질문

### 조직 시스템의 붕괴
- 50편까지는 관리 가능
- 500편부터 시스템 붕괴 — 검색이나 기억에 의존
- 프로젝트 시작 시 깔끔 → 쌓이면 포기

### 읽기 전략
- 선형 읽기가 아닌 전략적 읽기: 초록 → 결론 → 결과 → (필요 시) 방법론
- 같은 논문을 다른 깊이로 여러 번 읽음
- 전문가일수록 메모를 적게 하지만 자주 확인

## 핵심 고충 요약

1. **"내가 뭘 모르는지 모른다"** — 새 분야 진입 시 키워드 부재
2. **용어 변이** — 같은 개념, 다른 이름 (38편에서 135개 키워드)
3. **맥락 소실** — 왜 저장했는지 잊음
4. **중복 검색** — 이미 찾은 논문을 다시 찾음
5. **"끝"을 모름** — 리뷰 완료 기준 부재
6. **트리아지 부담** — 200편에서 10편 고르기
7. **도구 분산** — DB 5개, 프리프린트, 학회, SNS 통합 뷰 없음
