# 세션 11 — UI 리서치 + Living Notebook 재설계 (2026-02-23)

## 요약
Light House의 채팅 UI를 "Living Notebook" 패러다임으로 전환하기 위한 UI 리서치와 설계를 수행했다. 8가지 UI 방향을 탐색하고 5개 조합을 도출한 뒤, 사용자가 Combination 4(Living Notebook)를 기반으로 Board + Timeline/Graph를 결합하는 방향을 선택. 코딩 에이전트에게 구현을 위임하여 노트북 코어 및 UI 적용을 완료했다.

## 주요 활동

### 1. UI 방향 탐색 (8가지)
- Spatial Canvas, Graph/Network, Timeline/Journey, Split/Multi-pane, Card/Kanban, Ambient/Peripheral, Conversational Canvas, Novel AI Patterns
- 각 방향의 핵심 철학, 대표 제품, 장단점 분석

### 2. 조합 설계 (5가지)
1. Research Lab — 공간 캔버스 + 앰비언트 동료
2. Research Trail — 타임라인 + 분기 대화 + 지식 그래프
3. Reading Room — 3패널 + 칸반 + 능동적 사이드바
4. **Living Notebook** — 대화형 문서 + 지식 그래프 오버레이 + 생성형 UI ← 선택
5. Constellation — 별자리 지도 + 앰비언트 AI

### 3. 설계 결정
- 기본: Living Notebook + Board(3번에서) + Timeline/Graph(2번에서)
- 레이아웃: C 타입 3단 분할 (노트북 + Board 우상 + Graph 우하)
- 좌측 사이드바 → 상단 내비게이션 바 (세션 드롭다운)
- Board: 세로 칸반 4단계
- Graph: 타임라인 + 네트워크 서브뷰

### 4. 구현 위임 및 완료
- 노트북 코어 구현 완료 (코딩 에이전트)
- Board + Graph 추가 구현 계획 작성 완료
- UI 전체 적용 완료, 다음은 안정화 작업
