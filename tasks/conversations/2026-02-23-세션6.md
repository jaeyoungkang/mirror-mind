# 세션 6 — 2026-02-23

## 주제
Light House 서비스 설계 문서 재구성

## 요약
lighthouse 프로젝트의 대원칙(service-principles.md)이 최근 추가된 이후, 기존 서비스 설계 문서들이 정합성을 잃은 상태였다. 세 가지 구조적 문제(계층 역전, SPEC↔architecture 중복, 범용 원칙 산재)를 해결하기 위해 문서 체계를 처음부터 재구성했다.

## 핵심 작업

### 1. 문서 구조 검토 및 문제 진단
- lighthouse docs/ 8개 문서 전체를 탐색 에이전트로 검토
- 계층 역전: service-principles.md(대원칙)가 ai-design-framework.md(하위)에 핵심 정의를 위임
- SPEC.md 하단과 architecture.md의 대규모 중복 발견
- 범용 원칙(3-Plane, 웹 계약, 자율성 정의)이 여러 문서에 반복 정의

### 2. 재구성 계획 수립
- 새 구조: WHY/WHAT/HOW/CHECK/WATCH/RULES 역할 분리
- ai-design-framework.md → service-principles.md에 흡수 + 삭제 결정
- SPEC.md에서 기술 구현 내용을 architecture.md로 이관

### 3. 문서 재작성 실행
- service-principles.md: ai-design-framework.md 전체 흡수 (자율성 5단계, 설계 규칙, 안전망, 제어감, 신뢰도, 체크리스트)
- ai-design-framework.md: 삭제
- SPEC.md: 제품 스펙만 남기고 기술 내용 제거
- architecture.md: SPEC에서 이관된 기술 내용 흡수, 외부 참조 제거
- 하위 문서 참조 통일, AGENT.md 업데이트

### 4. 정합성 검증
- 탐색 에이전트로 전체 문서 교차 검증
- 5가지 기준 모두 통과 (범위 준수, 중복 없음, 참조 유효, 외부 참조 제거, 계층 일치)
